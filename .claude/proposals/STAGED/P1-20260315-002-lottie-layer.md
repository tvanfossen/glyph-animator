# Session 2: Lottie Layer

## Implementation Log

### Completed

- `lottie/easing.py` — EasingCurve ABC + Linear, EaseInOut, Organic, Elastic, Custom
  - 1D handles: `{x:[v], y:[v]}` for opacity, rotation, shape morph
  - 2D handles: `{x:[v1,v2], y:[v1,v2]}` for position, scale
- `lottie/paths.py` — `contour_to_lottie_path` + `lottie_path_to_contour` (round-trip exact)
  - Tangents are RELATIVE to vertex, not absolute
  - `offset_contour` for font→canvas coordinate transform
- `lottie/keyframes.py` — Factory functions for all property types:
  - `position_keyframe`: requires `to`/`ti` spatial tangents, 2D easing
  - `shape_keyframe`: `s`/`e` are arrays of path objects, 1D easing
  - `opacity_keyframe`/`rotation_keyframe`: 1D scalar, NO `to`/`ti`
  - `scale_keyframe`: 2D easing, NO `to`/`ti`
- `lottie/builder.py` — LottieBuilder: file structure, shape layers, groups, fills, strokes
  - Layer order: index 0 = FRONT
  - Group `tr` must be last in `it` array
- `lottie/constraints.py` — HardwareValidator: checks layers, shapes/layer, keyframes, file size
- `renderer/base.py` — RenderedDigit model (fitted_contours, outline_layers, matte_contour, arc_samples)
- `renderer/digit_renderer.py` — DigitRenderer: outline stack, matte (largest area), arc sampling
- `docs/LOTTIE_REFERENCE.md` — 13-section format reference with JSON examples + compatibility matrix
- `scripts/generate_morph_lottie.py` — morph Lottie generation (4 digit pairs)
- `scripts/generate_digit_outline.py` — digit with 3-layer outline + fill
- `scripts/build_viewer.py` — auto-discovers all JSON, regenerates viewer.html
- 33 new tests (82 total), all passing

### Stop Points Verified

1. Easing + paths: 13 tests confirm 1D/2D handle formats, path round-trip exact
2. Builder: morph Lottie (3→8, 0→8, 4→9, 2→5) verified playing in lottie-web viewer
3. Renderer: digit outlines generated as Lottie, viewer rebuilt with all files

### Design Decisions

- **Separate `build_viewer.py`**: Viewer generation decoupled from Lottie generation scripts. Discovers all JSONs in output dir, regenerates viewer.html. Avoids each script needing its own viewer logic.
- **Free functions in builder**: `static_val`, `animated_val`, `shape_group`, `animated_shape` etc. are module-level for ergonomics. LottieBuilder is for file-level orchestration.
- **No trim keyframes yet**: Planned in proposal but deferred — not needed until vine_clipped style (Session 3). The factory pattern makes adding them trivial.

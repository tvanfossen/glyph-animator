# Session 4: LVGL Desktop Viewer

## Scope

- `viewer/CMakeLists.txt` + LVGL as submodule (NOT ESP32 — native desktop SDL backend)
- `viewer/main.c` — window, file browser, layout
- `viewer/lottie_player.c` — playback (play/pause/step/scrub/speed)
- `viewer/config_panel.c` — parameter sliders, style selector
- `viewer/file_watcher.c` — auto-reload on output change
- Two playback modes: **creation** (appear from nothing) and **transition** (A→B)
- Performance overlay: frame render time, layer/shape count
- Build for Linux + macOS
- GitHub release binaries

## STOP Points (user verification required)

1. **After basic player**: loads and plays a Lottie file — **user verifies**
2. **After creation/transition toggle**: both modes work — **user verifies**
3. **After config panel**: parameter changes trigger regeneration — **user verifies**

## Acceptance Criteria

- Single binary, no runtime dependencies beyond SDL2
- Renders Lottie identically to lottie-web (same ThorVG backend as LVGL)
- Creation and transition modes clearly distinguished in UI
- Auto-reload works (file change → viewer updates)
- Performance overlay shows frame time for hardware viability assessment

## Playback Modes

Three distinct playback modes:
- **Creation**: loads `creation_3.json`, plays from empty canvas to fully formed glyph
- **Destruction**: loads `destruction_3.json`, plays from fully formed to empty
- **Transition**: loads `transition_3_8.json`, plays A fully formed → morphing → B fully formed

UI toggle switches between modes. Dropdown selects glyph(s).

## Architecture

```
viewer/
├── CMakeLists.txt           # LVGL as submodule, SDL2 backend
├── main.c                   # Window, file browser, layout
├── lottie_player.c          # Play/pause, frame step, scrub, speed
├── config_panel.c           # Real-time parameter sliders
├── file_watcher.c           # inotify/kqueue auto-reload
├── lv_conf.h                # LVGL config for desktop rendering
└── README.md
```

## Integration with Python Library

The viewer consumes Lottie JSON output from the Python library. Workflow:
1. Python CLI generates Lottie JSON to `output/` directory
2. Viewer watches `output/` directory
3. File change → viewer auto-reloads and replays
4. Config panel can invoke Python CLI with modified parameters

## Build Requirements

- CMake 3.16+
- SDL2 development libraries
- LVGL v9.3 (git submodule)
- ThorVG (bundled with LVGL)
- C11 compiler (gcc/clang)

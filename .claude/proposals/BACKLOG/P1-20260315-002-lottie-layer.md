## Key Lottie Format Details

These were discovered during prototype development and MUST be preserved:

- **Layer order**: index 0 = front (topmost visual layer)
- **Position keyframes**: require `to`/`ti` spatial tangents + 2D easing handles
- **Shape morph keyframes**: require `s` + `e` + easing handles
- **Scale/rotation keyframes**: NO `to`/`ti` — these are 1D properties
- **Path format**: `v` (vertices), `i` (in-tangents relative to vertex), `o` (out-tangents relative to vertex), `c` (closed)
- **Easing handles**: `[x, y]` for 1D properties, `[x1, x2]` / `[y1, y2]` for 2D
- **Tangent gotcha**: in/out tangents are RELATIVE to their vertex, not absolute coordinates

## Renderer ↔ Style Separation

```
Pipeline                    Renderer                   Style
(font → contours)    →    (contours → RenderedDigit)  →  (RenderedDigit → styled Lottie layers)
                           │                              │
                           ├── fitted_contours             ├── _build_creation(rendered)
                           ├── outline_layers              ├── _build_destruction(rendered)
                           ├── matte_layer                 └── _build_transition(rendered_a, rendered_b, pairs)
                           └── arc_samples
```

Renderer produces structural layers. Style adds content on top. Adding a new style never touches the renderer.

### Design Decisions

- **Constants from YAML, not Python**: `docs/constants.md` YAML frontmatter is the single source of truth. `constants.py` is a thin loader, not a definition file. This supports the Jinja2 resolution chain planned for Session 2+3.
- **`_ContourSet` helper class**: Introduced in `matching.py` to bundle contours + centroids + areas, avoiding functions with >6 arguments (flake8-functions limit).
- **`arc_length_inverse` refactored**: Extracted `_newton_solve` to stay within max-3-returns and max-6-args limits.
- **Free functions alongside Algorithm classes**: Algorithms implement the ABC for formal contract, but pipeline code calls free functions directly for ergonomics. Both share the same implementation.

### Known Limitations (addressed in later sessions)

- **Morph quality for dissimilar glyphs** (e.g. 1→7): Direct contour interpolation produces self-intersections when shapes are topologically different. Waypoint morphing (§7b) and non-morph styles (floral, vine, comic) address this in Session 3.
- **No Jinja2 config resolution yet**: Constants load from YAML, but the full chain (constants → proofs → styles → generation config) with Jinja2 template resolution is deferred to Session 2 (`config/manager.py`).
- **No Lottie output yet**: Pipeline produces matched contours, not Lottie JSON. Session 2 adds the Lottie builder.

## Reference: Algorithm ABC Contract

```python
class Algorithm(ABC, Generic[TInput, TOutput]):
    @property
    @abstractmethod
    def proof_reference(self) -> str: ...

    @property
    @abstractmethod
    def complexity(self) -> str: ...

    @property
    @abstractmethod
    def error_bound(self) -> str: ...

    @abstractmethod
    def execute(self, input_data: TInput) -> TOutput: ...
```

## Reference: Configuration Architecture (3 layers + Jinja2 resolution)

```
docs/constants.md          → base namespace (PHI, GOLDEN_ANGLE, Gauss nodes, etc.)
    ↓
docs/proofs/*.md           → algorithm tunables, can reference {{constants.*}}
    ↓
docs/styles/*.md           → style config, can reference {{constants.*}} and {{proofs.*}}
    ↓
configs/*.yaml             → generation config, can override any value from above
```
